📝 INPUT (사람이 적음)
Schema (TypeQL 풍)
define

한옥카페 sub 한옥;
한옥     sub 전통건축물;
전통건축물 sub 건물;
건물     sub 자산;

adjacent sub relation,
  symmetric;
Rules
rule fire-risk-dispatch:
  when {
    $b isa 전통건축물;
    $b has fire_risk >= 0.8;
  } then {
    dispatch(폼차, 3);
  };
Facts (insert)
insert
$b1 isa 한옥카페;
$b2 isa 한옥;
(b1, b2) isa adjacent;
$b1 has fire_risk 0.85;
총 4줄 + α. 이게 사람이 적는 모든 것. 나머지는 reasoner가 알아서.

    💡 발견

    네 줄(+ 룰 한 개)을 적었더니, reasoner가 그 안에서 20개 안팎의 함축을 끝까지 끌어냈습니다. 이게 *추론구조의 실체*예요. 컴퓨터가 우리를 위해 판단해주는 게 아니라 — 우리가 미리 명제로 적어둔 구조를 따라 함축이 자동으로 펼쳐진 것일 뿐입니다.

    적은 것은 적게 적었지만, 그 적은 것 안에 잠긴 결론은 작지 않습니다. 우리가 적어둔 것과 적어둔 것에서 따라오는 것은 다른 크기의 집합이고, reasoner는 후자를 끝까지 끌어내는 도구일 뿐입니다.
